 <span data-ttu-id="733f9-104">**Résumé :** Comprendre comment l'adoption cloud nécessite une nouvelle approche des investissements relatifs à l'infrastructure réseau.</span><span class="sxs-lookup"><span data-stu-id="733f9-104">**Summary:** Understand how cloud adoption requires a new approach to network infrastructure investments.</span></span>
  
<span data-ttu-id="733f9-p101">La migration vers le cloud modifie le volume et la nature des flux de trafic internes et externes à un réseau d’entreprise. Elle a également une incidence sur les approches permettant de réduire les risques de sécurité.</span><span class="sxs-lookup"><span data-stu-id="733f9-p101">Cloud migration changes the volume and nature of traffic flows within and outside a corporate network. It also affects approaches to mitigating security risk.</span></span>
  
- <span data-ttu-id="733f9-107">Avant le cloud</span><span class="sxs-lookup"><span data-stu-id="733f9-107">Before the cloud</span></span>
    
    <span data-ttu-id="733f9-p102">La plupart des investissements concernant les infrastructures réseau visaient à garantir la disponibilité, la fiabilité et les performances de la connectivité pour les centres de données locaux. Les limites de réseau constituaient la principale défense contre les violations de sécurité.</span><span class="sxs-lookup"><span data-stu-id="733f9-p102">Most networking infrastructure investments were spent on ensuring available, reliable, and performant connectivity to on-premises datacenters. - <span data-ttu-id="733f9-111">Après le cloud</span><span class="sxs-lookup"><span data-stu-id="733f9-111">After the cloud</span></span>
    
    <span data-ttu-id="733f9-p103">Avec la migration de la nouvelle productivité et les charges de travail informatiques exécutées dans le cloud, les investissements concernant les infrastructures ont délaissé les centres de données locaux au profit de la connectivité Internet, qui est désormais essentielle pour les opérations d’entreprise internes. La connectivité fédérée oriente la stratégie de sécurité sur la protection des identités et des données lors de leur circulation sur le réseau et sur les points de connectivité aux services cloud de Microsoft.</span><span class="sxs-lookup"><span data-stu-id="733f9-p103">With new and migrated productivity and IT workloads running in the cloud, infrastructure investments shift from on-premises datacenters to Internet connectivity, which is now critical for internal business operations. Federated connectivity shifts security strategy to protecting identities and data as they flow through the network and points of connectivity to Microsoft cloud services.</span></span>
    
<span data-ttu-id="733f9-p104">Investissements concernant les infrastructures réseau commencent par la connectivité. Les investissements supplémentaires dépendent de la catégorie du service cloud.</span><span class="sxs-lookup"><span data-stu-id="733f9-p104">Network infrastructure investments begin with connectivity. Additional investments depend on the category of cloud service.</span></span>
  
- <span data-ttu-id="733f9-p105">**Software as a Service (SaaS)** les services SaaS de Microsoft incluent Office 365, Microsoft Intune et Microsoft Dynamics 365.     <span data-ttu-id="733f9-p106">L’architecture réseau se concentre sur une connectivité fiable et redondante, ainsi que sur une bande passante suffisante. Les investissements en cours incluent la surveillance et l’optimisation des performances.</span><span class="sxs-lookup"><span data-stu-id="733f9-p106">Network architecture focuses on reliable, redundant connectivity and ample bandwidth. - <span data-ttu-id="733f9-p107">**Azure Platform as a Service (PaaS)**: en plus des investissements pour les services SaaS de Microsoft, les applications PaaS pour plusieurs sites ou réparties géographiquement peuvent nécessiter la création d'un système Azure Traffic Manager pour distribuer le trafic client. Ongoing investments include performance and traffic distribution monitoring and failover testing.</span></span>
    
- <span data-ttu-id="733f9-p108">**Azure Infrastructure as a Service (IaaS)**: en plus des investissements pour les services SaaS et PaaS de Microsoft, l'exécution de charges de travail informatiques dans IaaS nécessite la création et la configuration de réseaux virtuels Azure hébergeant des machines virtuelles, la connectivité sécurisée des applications exécutées sur ces dernières, le routage, l'adressage IP, le DNS et l'équilibrage de charge. Les investissements en cours incluent la surveillance et le dépannage des performances et de la sécurité.</span><span class="sxs-lookup"><span data-stu-id="733f9-p108">**Azure Infrastructure as a Service (IaaS)** In addition to the investments for Microsoft SaaS and PaaS services, running IT workloads in IaaS requires the design and configuration of Azure virtual networks that host virtual machines, secure connectivity to applications running on them, routing, IP addressing, DNS, and load balancing. Ongoing investments include performance and security monitoring and troubleshooting.</span></span>

<span data-ttu-id="733f9-124">[Microsoft 365](https://www.microsoft.com/microsoft-365) est une combinaison d'Office 365, Enterprise Management + Security (EMS) et Windows 10.</span><span class="sxs-lookup"><span data-stu-id="733f9-124">[Microsoft 365](https://www.microsoft.com/microsoft-365) is a combination of Office 365, Enterprise Management + Security (EMS), and Windows 10.</span></span> <span data-ttu-id="733f9-125">Microsoft 365 combine plusieurs services SaaS et Azure pour une solution complète et intelligente qui permet à chacun d'être créatif et de collaborer en toute sécurité.</span><span class="sxs-lookup"><span data-stu-id="733f9-125">Microsoft 365 combines multiple SaaS and Azure services for a complete, intelligent solution that empowers everyone to be creative and work together securely.</span></span>
